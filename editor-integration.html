外部编辑器接入指南External Editor Integration

Usuzumi 提供编辑器外壳、源码区、预览区、轻量 Markdown 预览和事件桥接；本站不再携带站点专用编辑器运行时。 Usuzumi provides editor shells, source regions, preview regions, light Markdown preview, and event bridges; this site no longer ships site-owned editor runtime.

边界Boundary

外壳属于 Usuzumi，编辑能力属于项目Usuzumi owns the shell; applications own editing capability.

能力Capability Usuzumi 提供Provided by Usuzumi 项目接入Application integration
Markdown .uzu-markdown-editor、data-uzu-markdown-source、data-uzu-markdown-preview 本站使用 Usuzumi 公共运行时渲染轻量预览。This site renders light previews with the public Usuzumi runtime.
短代码输入Short code input textarea.uzu-code-editor 使用原生 textarea 事件保存内容。Use native textarea events to save content.
通用编辑器外壳Generic editor shell .uzu-editor、.uzu-editor-toolbar、.uzu-editor-surface 项目自行选择编辑器并挂载到外壳内。Applications choose and mount their own editor engines inside the shell.

Markdown

Markdown：保留源码区和预览区Markdown: keep source and preview separate.

运行时边界Runtime boundary

组件页只加载 Usuzumi 公共 CSS 和运行时。需要更完整的 Markdown 策略时，业务项目可以在 Usuzumi 外壳上接入自己的解析器。The component page only loads public Usuzumi CSS and runtime. Applications can connect their own parser to the Usuzumi shell when they need a fuller Markdown policy.

<section class="uzu-editor uzu-markdown-editor" data-uzu-markdown-editor data-uzu-markdown-render>
  <textarea class="uzu-markdown-source" data-uzu-markdown-source># Draft</textarea>
  <div class="uzu-markdown-preview uzu-prose" data-uzu-markdown-preview></div>
</section>
const shell = document.querySelector('[data-uzu-markdown-editor]');
const source = shell.querySelector('[data-uzu-markdown-source]');
const preview = shell.querySelector('[data-uzu-markdown-preview]');

function render(value) {
  preview.replaceChildren(window.Usuzumi.renderMarkdown(value));
  shell.dispatchEvent(new CustomEvent('uzu-markdown-editor-render', {
    bubbles: true,
    detail: { editor: shell, source, preview, value }
  }));
}

shell.addEventListener('uzu-markdown-editor-change', (event) => {
  render(event.detail.value);
});

render(source.value);
