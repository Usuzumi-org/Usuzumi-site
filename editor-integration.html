Guide

外部编辑器接入指南External Editor Integration

Usuzumi 负责编辑区外观、工具栏按钮和事件桥。文档模型、快捷键、历史记录、语法服务和完整 Markdown 解析交给选定的编辑器库：Tiptap、markdown-it 和 CodeMirror 6。 Usuzumi owns the editor surface, toolbar buttons, and event bridge. Document models, shortcuts, history, language services, and full Markdown parsing stay in the chosen editor libraries: Tiptap, markdown-it, and CodeMirror 6.

边界Boundary

先决定谁负责哪一层。Decide which layer owns what.

能力Capability UsuzumiUsuzumi 外部库External library
富文本Rich text .uzu-editor、工具栏、uzu-editor-command Tiptap 管理 schema、selection、history 和命令执行。Tiptap owns schema, selection, history, and commands.
MarkdownMarkdown .uzu-markdown-editor、源码区、预览区、change/render 事件 markdown-it 管 Markdown 渲染和安全策略。markdown-it owns Markdown rendering and safety policy.
代码编辑Code editing .uzu-code-editor 或 .uzu-editor 外壳 CodeMirror 6 提供高亮、补全、诊断和多光标。CodeMirror 6 provides highlighting, completion, diagnostics, and multi-cursor editing.

Rich text

富文本：把工具栏命令交给编辑器实例。Rich text: pass toolbar commands to the editor instance.

1. HTML

外层加 data-uzu-rich-editor，工具栏按钮声明命令名。Add data-uzu-rich-editor and declare command names on toolbar buttons.

2. Mount

把 Tiptap 挂到 data-uzu-editor-surface。Mount Tiptap into data-uzu-editor-surface.

3. Events

监听 uzu-editor-command，根据 event.detail.command 执行命令。Listen for uzu-editor-command and execute by event.detail.command.

4. State

选区变化时同步 aria-pressed 和 .is-active。Sync aria-pressed and .is-active when selection changes.

<section class="uzu-editor" data-uzu-rich-editor>
  <div class="uzu-editor-toolbar" role="toolbar" aria-label="Text tools">
    <button class="uzu-toolbar-button" type="button" data-uzu-editor-command="bold" data-uzu-editor-toggle>B</button>
    <button class="uzu-toolbar-button" type="button" data-uzu-editor-command="italic" data-uzu-editor-toggle>I</button>
    <button class="uzu-toolbar-button" type="button" data-uzu-editor-command="heading" data-uzu-editor-value="2">H2</button>
  </div>
  <div class="uzu-editor-surface" data-uzu-editor-surface></div>
</section>
const shell = document.querySelector('[data-uzu-rich-editor]');
const surface = shell.querySelector('[data-uzu-editor-surface]');

// Example shape: replace this with your Tiptap instance.
const editor = createRichTextEditor({
  element: surface,
  content: '<p>Start writing.</p>',
  onUpdate: ({ html }) => {
    shell.dispatchEvent(new CustomEvent('uzu-editor-change', {
      bubbles: true,
      detail: { editor: shell, surface, value: html }
    }));
  }
});

shell.addEventListener('uzu-editor-command', (event) => {
  const { button, command, value } = event.detail;
  if (command === 'bold') editor.chain().focus().toggleBold().run();
  if (command === 'italic') editor.chain().focus().toggleItalic().run();
  if (command === 'heading') editor.chain().focus().toggleHeading({ level: Number(value) }).run();
  button.setAttribute('aria-pressed', editor.isActive(command) ? 'true' : 'false');
  button.classList.toggle('is-active', editor.isActive(command));
});

Markdown

Markdown：源码编辑和预览渲染分开接。Markdown: wire source editing and preview rendering separately.

内置渲染器的定位Built-in renderer scope

data-uzu-markdown-render 只适合轻量预览。需要表格、脚注、语法扩展、清洗策略或服务端一致输出时，把预览交给项目自己的 Markdown 管线。data-uzu-markdown-render is for light previews. Use markdown-it for tables, plugins, sanitization, and server/client parity.

<section class="uzu-editor uzu-markdown-editor" data-uzu-markdown-editor>
  <textarea class="uzu-markdown-source" data-uzu-markdown-source># Draft</textarea>
  <div class="uzu-markdown-preview uzu-prose" data-uzu-markdown-preview></div>
</section>
const shell = document.querySelector('[data-uzu-markdown-editor]');
const source = shell.querySelector('[data-uzu-markdown-source]');
const preview = shell.querySelector('[data-uzu-markdown-preview]');

// markdown-it renders the preview from the source value.
function render(value) {
  preview.innerHTML = markdownIt.render(value);
  shell.dispatchEvent(new CustomEvent('uzu-markdown-editor-render', {
    bubbles: true,
    detail: { editor: shell, source, preview, value }
  }));
}

shell.addEventListener('uzu-markdown-editor-change', (event) => {
  render(event.detail.value);
});

render(source.value);

Code editor

代码编辑：用 Usuzumi 定边界，用编辑器库接能力。Code editing: use Usuzumi for the boundary and an editor library for capability.

短内容Short content

配置片段、命令文本和小段代码可以直接用 textarea.uzu-code-editor。Use textarea.uzu-code-editor for configuration snippets, command text, and short code.

长内容Long content

语法高亮、补全、诊断和大文件滚动交给 CodeMirror 6，再把外层放进 .uzu-editor。Let CodeMirror 6 handle highlighting, completion, diagnostics, and large-file scrolling, then place it inside .uzu-editor.
